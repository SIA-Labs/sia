# SIA Framework Uninstallation Guide

This guide provides manual instructions for removing the SIA framework from your project.

> **Note**: These are manual steps. There is no automated uninstall script to prevent accidental deletion of your work.

---

## ‚ö†Ô∏è Before You Begin

**Warning**: Uninstalling SIA will remove:
- All agents, knowledge base, requirements, and skills in `.sia/`
- SIA framework files in `sia/` submodule
- Copilot instructions in `.github/copilot-instructions.md`
- Auto-detected configuration in `.sia.detected.yaml`

**Backup**: Consider creating a backup before proceeding:
```bash
# Create backup
cp -r .sia .sia.backup
cp .github/copilot-instructions.md .github/copilot-instructions.md.backup
cp .sia.detected.yaml .sia.detected.yaml.backup
```

## üóëÔ∏è Uninstallation Steps

### Step 1: Remove SIA Submodule

```bash
# Remove submodule from git configuration
git submodule deinit -f sia

# Remove submodule from .git/modules
rm -rf .git/modules/sia

# Remove submodule directory
git rm -f sia

# Commit the removal
git commit -m "chore: Remove SIA framework submodule"
```

### Step 2: Remove Project-Specific SIA Files

```bash
# Remove .sia directory (agents, knowledge, requirements, skills)
rm -rf .sia

# Remove auto-detected configuration
rm -f .sia.detected.yaml

# Remove Copilot instructions (if generated by SIA)
rm -f .github/copilot-instructions.md
```

### Step 3: Clean Git Configuration

```bash
# Remove .gitmodules file if SIA was the only submodule
# (Check first if you have other submodules)
cat .gitmodules

# If empty or only contains SIA, remove it
rm -f .gitmodules
```

### Step 4: Commit Changes

```bash
# Stage all deletions
git add -A

# Commit the cleanup
git commit -m "chore: Remove SIA framework and related files"

# Push to remote
git push origin main
```

## üîç Verification

After uninstallation, verify that all SIA components are removed:

```bash
# Check for remaining SIA files
find . -name ".sia*" -o -name "sia"

# Check git submodules
git submodule status

# Check .gitmodules (should not exist or not contain SIA)
cat .gitmodules
```

Expected output: No SIA-related files or submodules found.

## üîÑ Reinstallation

If you want to reinstall SIA later:

```bash

# Run installer
cd sia && bash installer/install.sh
```

## ‚ùì Frequently Asked Questions

### Can I remove just the `.sia/` directory and keep the framework?

Yes, but not recommended. The `.sia/` directory contains project-specific configuration. If you only want to reset your configuration:

```bash
# Backup your work
mv .sia .sia.old

# Reinstall structure
cd sia && bash installer/install.sh

# Ask Copilot to reinitialize
# "Initialize SIA for this repository"
```

### What if I only want to remove archived files?

```bash
# Remove archived knowledge
rm -rf .sia/knowledge/_archive/*

# Remove archived requirements
rm -rf .sia/requirements/_archive/*

# Commit cleanup
git add .sia
git commit -m "chore: Clean archived SIA files"
```

### Can I keep the framework but disable Copilot integration?

Yes, simply remove or rename the Copilot instructions:

```bash
# Rename instead of delete (keeps backup)
mv .github/copilot-instructions.md .github/copilot-instructions.md.disabled

# Commit
git add .github
git commit -m "chore: Disable SIA Copilot integration"
```

## üÜò Troubleshooting

### Error: "fatal: not a git repository"

If you're trying to uninstall in a non-git directory, manually remove the files:

```bash
rm -rf sia .sia .sia.detected.yaml
rm -f .github/copilot-instructions.md
```

### Error: "No submodule mapping found"

If the submodule is already partially removed:

```bash
# Force remove directory
rm -rf sia

# Clean git index
git rm --cached sia

# Remove from .gitmodules
# Edit .gitmodules and remove [submodule "sia"] section

# Commit
git add -A
git commit -m "chore: Complete SIA removal"
```

### I accidentally deleted files, can I recover?

If you haven't committed yet:

```bash
# Restore from git
git restore sia .sia .sia.detected.yaml .github/copilot-instructions.md
```

If you already committed, you can recover from your backup or from the previous commit:

```bash
# Restore from previous commit
git checkout HEAD~1 -- sia .sia .sia.detected.yaml .github/copilot-instructions.md
```
